# docs/ai — durable "why" reasoning

**Committed. Permanent.** This is where the reasoning behind decisions lives when it's too long for a
one-line `decisions.md` entry — the narrative of *why* the system is the way it is.

Principle 3: **code answers _how_; docs answer _why_.** Don't describe how the code works here (the code
does that, and prose drifts out of date). Describe *why* it works that way — the constraints, the
trade-offs, the context a future reader won't have.

One file per topic. Link to it from the relevant `decisions.md` entry.

> This is a **growth** folder, not part of the minimal kernel. Create it when a decision needs more than a
> ledger line to explain. Delete this README once you add your first real doc.
