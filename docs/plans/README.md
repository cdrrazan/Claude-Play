# docs/plans — ephemeral implementation notes

**Scratchpad. Deleted after use.** Working notes for an in-flight change — the step-by-step an agent (or
you) is following right now. Unlike `docs/ai/`, nothing here is meant to last.

**Delete a plan once its work graduates** (merged and verified). If you don't clean these up, they rot into
misleading half-truths that contradict the code. A stale plan is worse than no plan.

If a plan contains reasoning worth keeping, promote that reasoning into `docs/ai/` or a `decisions.md`
entry *before* you delete the plan — then delete it.

> Growth folder, not kernel. Delete this README once you add your first real plan.
